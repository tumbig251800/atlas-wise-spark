# ATLAS Logic — กฎการวิเคราะห์และ Compassion Protocol

> ไฟล์นี้คือ "กฎหลัก" ของระบบ ATLAS
> Agent ต้องอ่านไฟล์นี้ก่อนวิเคราะห์ข้อมูลทุกครั้ง

---

## ส่วนที่ 1 — Gap Categories (ประเภทช่องว่างการเรียนรู้)

### K-Gap — ช่องว่างด้านความเข้าใจ / Concept

**นิยาม:** นักเรียนยังไม่เข้าใจหลักการ แนวคิด หรือทฤษฎีที่จำเป็น

**สัญญาณที่พบบ่อย:**
- ทำโจทย์ได้บางส่วนแต่อธิบายไม่ได้
- ตอบคำถามปากเปล่าผิดพลาดซ้ำๆ ในจุดเดิม
- สับสนระหว่าง concept ที่คล้ายกัน
- ครูบันทึกว่า "ยังไม่เข้าใจ concept", "ต้องอธิบายซ้ำ", "งงในส่วนของทฤษฎี"

**แนวทางซ่อมเสริม:** อธิบายใหม่ด้วยวิธีที่ต่างออกไป ใช้ตัวอย่างจากชีวิตจริง สร้าง concept map

---

### P-Gap — ช่องว่างด้านทักษะ / Practice

**นิยาม:** นักเรียนเข้าใจหลักการแล้ว แต่ยังทำเองไม่คล่อง ต้องการฝึกเพิ่ม

**สัญญาณที่พบบ่อย:**
- ทำถูกเมื่อมีคนช่วย แต่ทำคนเดียวผิด
- ทำช้า ไม่คล่อง หรือต้องดูตัวอย่างตลอด
- ครูบันทึกว่า "ต้องฝึกเพิ่ม", "ทำได้แต่ช้า", "ต้องมีคนกำกับ"

**แนวทางซ่อมเสริม:** เพิ่มแบบฝึกหัด ฝึกซ้ำ peer practice

---

### A-Gap — ช่องว่างด้านเจตคติ / Engagement (ระดับ 1)

> **A-Gap และ A2-Gap เป็นปัญหาด้านเจตคติ/พฤติกรรมเหมือนกัน แต่ต่างกันที่ความรุนแรง**
> - A-Gap = ยังจัดการได้ในห้องเรียน → ครูแก้ไขเองได้
> - A2-Gap = รุนแรงเกินขอบเขตห้องเรียน → ต้องส่งต่อ (Referral)

**นิยาม:** นักเรียนไม่ตั้งใจเรียน ไม่มีสมาธิ ขาดแรงจูงใจ หรือมีพฤติกรรมที่รบกวนการเรียน แต่ยังอยู่ในขอบเขตที่ครูจัดการได้ในห้องเรียนปกติ

**สัญญาณที่พบบ่อย:**
- ไม่ตั้งใจ ไม่มีสมาธิ ฝันกลางวัน เหม่อลอย
- ไม่ส่งงาน ไม่ตอบคำถาม ไม่มีส่วนร่วมในกิจกรรม
- ครูบันทึกว่า "ไม่ตั้งใจ", "ไม่ส่งงาน", "engagement ต่ำ", "ไม่มีแรงจูงใจ"

> ⚠️ ตรวจสอบก่อนเสมอว่าไม่ใช่ Special Care หรือ A2-Gap ที่ถูก label ผิด

**แนวทางซ่อมเสริม:** สร้างความสัมพันธ์ ค้นหาสาเหตุ ปรับวิธีสอนหรือสภาพแวดล้อม

---

### A2-Gap — พฤติกรรมก้าวร้าว / Referral Case (ระดับ 2)

> **A2-Gap คือ A-Gap ที่รุนแรงจนเกินขอบเขตที่ครูจัดการได้ในห้องเรียน → บังคับต้องส่งต่อ**

**นิยาม:** นักเรียนแสดงพฤติกรรมก้าวร้าว ทำร้ายผู้อื่น หรือมีพฤติกรรมที่ไม่สามารถจัดการได้ในห้องเรียนปกติ → ต้องส่งต่อ (Referral) ทันที

> ⚠️ **ห้ามสับสนสองสิ่งนี้ — ใช้ชื่อคล้ายกันแต่คนละระบบ:**
> - `a2-gap` ใน field `major_gap` = **พฤติกรรมก้าวร้าว (Referral)** ← KPA framework
> - สมรรถนะ `A2` ใน `unit_assessments` = **การคิดขั้นสูง** ← รหัสสมรรถนะหลักสูตร
> ทั้งสองไม่มีความเกี่ยวข้องกัน

**สัญญาณที่พบบ่อย:**
- นักเรียนทำร้ายเพื่อน ทำร้ายตัวเอง หรือทำลายสิ่งของ
- ครูบันทึกว่า "ควบคุมพฤติกรรมไม่ได้", "ก้าวร้าว", "เสี่ยงต่อความปลอดภัย"
- พฤติกรรมรุนแรงซ้ำต่อเนื่องจนห้องเรียนหยุดชะงัก

**การจัดการ:**
- หยุดกิจกรรมและดูแลความปลอดภัยทันที
- ส่งต่อครูแนะแนว / Welfare / ผู้บริหาร (Referral) — บังคับ ไม่ใช่ทางเลือก
- **ไม่นับเป็น Gap ด้านการเรียนรู้ของครู** และไม่ใช่ indicator ของสมรรถนะนักเรียน
- รายงานเฉพาะจำนวน — ห้ามระบุรายละเอียดนักเรียนในรายงานหลัก

---

### System-Gap — ช่องว่างด้านระบบ / Infrastructure

**นิยาม:** ปัญหาไม่ได้อยู่ที่นักเรียนหรือครู แต่อยู่ที่ระบบ เช่น ข้อมูลไม่ครบ อุปกรณ์ขัดข้อง หลักสูตรไม่เหมาะ

**สัญญาณที่พบบ่อย:**
- ข้อมูลใน log ไม่ครบหรือขัดแย้ง
- ครูบันทึกว่า "ระบบขัดข้อง", "ไม่มีอุปกรณ์", "หลักสูตรไม่ตรง"

**การจัดการ:** บันทึกเป็น System-Gap และระบุใน Integrity Audit

---

### Success — สำเร็จตามเป้าหมาย

**นิยาม:** นักเรียนบรรลุเป้าหมายการเรียนรู้ของหน่วย mastery_score >= 4 หรือ major_gap == "success"

**เงื่อนไข:**
- mastery_score >= 4 **และ** ไม่มี flag A-Gap หรือ A2-Gap
- หรือ major_gap field มีค่า "success" / "สำเร็จ" / "ผ่าน"

## ส่วนที่ 2 — Compassion Protocol

> โปรโตคอลนี้มีผลก่อน gap วิเคราะห์ทุกครั้ง

### 2.1 คำนิยาม

**Health Care** = นักเรียนที่มีความต้องการพิเศษด้านสุขภาพ เช่น โรคประจำตัว ภาวะทางการแพทย์ ที่ส่งผลต่อการเรียน

**Special Care** = นักเรียนที่ต้องการการดูแลพิเศษ ครอบคลุมทั้ง Health Care และกรณีอื่น เช่น สถานการณ์ครอบครัว ความต้องการพิเศษทางการศึกษา

### 2.2 กฎการแยก Special Care

```
ขั้นตอนบังคับ (ทำก่อนคำนวณ Gap ทุกครั้ง):

1. ตรวจ field health_care_status
   - ถ้า health_care_status == true / "yes" / "มี" / "Y"
     → นักเรียนกลุ่มนี้เป็น Special Care

2. ตรวจ field health_care_ids
   - ถ้ามีค่าในช่องนี้ (ไม่ว่างเปล่า)
     → log นี้เกี่ยวข้องกับนักเรียน Special Care

3. แยกออกจาก dataset หลัก
   - นำ log ที่เกี่ยวกับ Special Care ไปรายงานใน Special Care tab
   - ไม่นำไปรวมใน Gap calculation หลัก

4. นับจำนวน Special Care
   - รายงานเป็น [จำนวน X คน] เท่านั้น
   - ห้ามระบุชื่อในรายงานหลัก
```

### 2.3 กฎการรายงาน Special Care

- ✅ รายงานจำนวน (เช่น "Special Care 5 คน")
- ✅ ระบุ accommodation ที่ใช้ (เช่น "ปรับเวลาส่งงาน", "นั่งแถวหน้า")
- ✅ รายงาน mastery trend ของกลุ่มนี้แยกต่างหาก
- ❌ ห้ามระบุชื่อนักเรียนในรายงานหลัก
- ❌ ห้ามระบุรายละเอียดสุขภาพที่อ่อนไหว
- ❌ ห้ามนับเป็น A-Gap หรือความล้มเหลวของครู
- ❌ ห้ามเปรียบเทียบผลลัพธ์กับกลุ่มปกติโดยตรงในรายงานหลัก

### 2.4 Safety First Protocol

ถ้า log ระบุสัญญาณ A2-Gap หรือความเสี่ยงสูง:

```
1. ไม่วิเคราะห์ต่อในมิติการเรียน
2. บันทึกใน Special Care tab ว่า "ต้องการการดูแลเร่งด่วน"
3. แนะนำให้ผู้บริหารและครูแนะแนวรับทราบ
4. ไม่ระบุรายละเอียดในรายงานหลัก
```

## ส่วนที่ 3 — Integrity Audit Rules

> Integrity Audit คือการตรวจหาความขัดแย้งหรือข้อผิดพลาดในข้อมูล
> **ไม่ใช่การตำหนิครู** — เป็นการช่วยให้ข้อมูลน่าเชื่อถือ

### 3.1 Flags ที่ต้องตรวจ

| Flag ID | ชื่อ Flag | เงื่อนไข | ความรุนแรง |
|---------|---------|---------|-----------|
| INT-01 | Special Care ถูกนับเป็น A-Gap | health_care_status = true แต่ major_gap = "A-Gap" | สูง |
| INT-02 | ป่วย/พักแต่ถูกนับเป็น Remedial | health_care_status = true แต่มีใน remedial_ids | สูง |
| INT-03 | Special Care คะแนนสูงผิดปกติ | health_care = true และ mastery_score = 5 | ต่ำ (ตรวจบริบท) |
| INT-04 | ชื่อครูซ้ำหลายรูปแบบ | teacher_name มีรูปแบบคล้ายกันหลายรูปแบบ | กลาง |
| INT-05 | ข้อมูลดิบในช่อง reflection | reflection หรือ key_issue มีรหัส/ข้อมูลระบบ | ต่ำ |
| INT-06 | คะแนนนอกช่วง | mastery_score < 1 หรือ > 5 | สูง |
| INT-07 | วันที่ไม่สอดคล้อง | teaching_date อยู่นอกช่วงที่กำหนดในรายงาน | กลาง |
| INT-08 | Log ซ้ำ | teacher_id + teaching_date + subject ซ้ำกัน | กลาง |
| INT-09 | major_gap ว่างเปล่า | major_gap เป็น null หรือ empty string | สูง |
| INT-10 | Remedial ไม่มี log ซ่อมเสริม | remedial_ids มีค่า แต่ไม่มี follow-up log | ต่ำ |

### 3.2 วิธีรายงาน Flag

```
สำหรับแต่ละ flag ให้บันทึกใน Integrity Audit tab:
  - Flag ID
  - Flag ชื่อ
  - จำนวน records ที่พบ
  - ตัวอย่าง 1–3 records
  - คำแนะนำ (เช่น "ควรตรวจสอบกับครู" ไม่ใช่ "ครูทำผิด")
  - ระดับความรุนแรง: สูง / กลาง / ต่ำ
```

### 3.3 สิ่งที่ห้ามทำในรายงาน

- ❌ ห้ามสรุปว่าครูทำผิดจาก flag เพียงอย่างเดียว
- ❌ ห้ามระบุชื่อนักเรียนใน Integrity Audit tab
- ❌ Flag INT-03 (Special Care คะแนนสูง) ห้ามสรุปว่าผิดปกติ — ให้ระบุว่า "ควรตรวจบริบทเพิ่มเติม"

## ส่วนที่ 4 — Red Zone และ Success Rate

### Red Zone

```
เงื่อนไข: mastery_score ≤ 2.5 (สเกล 1–5; ตาม Executive Dashboard / Trend Alerts ของ ATLAS)
ถ้าในฐานข้อมูลเป็นจำนวนเต็ม 1–5 เท่านั้น ค่าที่ถือเป็น Red Zone คือ 1 และ 2
หมายความว่า: คลาสนั้นมีความเสี่ยงสูงที่นักเรียนจะตามไม่ทัน
การรายงาน: นับจำนวน log ที่เป็น Red Zone ต่อ subject / ช่วงชั้น
สี: แดง (#dc2626) ในรายงาน
```

### Success Rate

```
เงื่อนไข: major_gap == "success" หรือ mastery_score >= 4
การคำนวณ:
  success_rate = (จำนวน log ที่เป็น Success / จำนวน log ทั้งหมดที่ไม่ใช่ Special Care) × 100
หน่วย: ร้อยละ (%)
```

### Yellow Zone

```
เงื่อนไข: mastery_score == 3
หมายความว่า: กำลังพัฒนา ต้องติดตาม
สี: เหลือง (#f59e0b) ในรายงาน
```

### Green Zone

```
เงื่อนไข: mastery_score >= 4
สี: เขียว (#22c55e) ในรายงาน
```
