# ฟีเจอร์ในแอป: แท็บ "เนื้อหา vs ประยุกต์" (PBL × คะแนนหลังหน่วย)

> เจ้าของงาน: ฝ่ายวิชาการ ATLAS (โรงเรียนวรนาถวิทยากำแพงเพชร)

## เป้าหมาย
เพิ่มแท็บที่ 4 ในหน้า PBL Dashboard (`/pbl`) ให้ครู **เปิดดูเองได้สดทุกเมื่อ** ว่าควรพัฒนาเด็กแต่ละคนไป
ทิศทางไหน โดยเชื่อมคะแนน **PBL (สมรรถนะข้ามวิชา 5 ด้าน)** กับ **คะแนนหลังหน่วย K/P/A (ความรู้รายวิชา)**

เป็นช่องทางที่ 3 ของชุดเดียวกัน (ตรรกะต้องตรงกันทั้งหมด):
- MCP tool `atlas_pbl_unit_crosscheck` (v2.5.0, live) — แหล่งข้อมูล/ใช้กับ AI/Codex
- รายงาน .docx รายครู (อยู่ใน `Desktop/Atlas-cowork  report/01_รายงาน/PBL_สมรรถนะผู้เรียน_2569-1/`) — เล่มรอประเมินภายนอก
- **แท็บในแอป (อันนี้)** — ครูดูเองสด ๆ

## ที่แก้
`src/pages/PBLDashboard.tsx` — เพิ่ม `<TabsTrigger value="crosscheck">เนื้อหา vs ประยุกต์</TabsTrigger>`
และ `<TabsContent value="crosscheck">` (ตอนนี้มี overview / student / classroom)

## ข้อมูล
- PBL: มี query `detail` ในไฟล์อยู่แล้ว (student_id, 5 คะแนน, overall_result, notes, project)
- หลังหน่วย: query `unit_assessments` ฝั่ง frontend ได้ (RLS เปิดให้ครู/แอดมิน — ฟีเจอร์ UnitScore อ่านอยู่แล้ว)
  - filter: `academic_term` = เทอมที่เลือก, `grade_level` + `classroom` ตามตัวกรอง
  - `unit% = (k_score+p_score+a_score) / (k_total+p_total+a_total) * 100` เฉลี่ยทุก row ของนักเรียนคนนั้น
- join ในแอปด้วย key `student_id + grade_level + classroom` (ภายในเทอม/ห้องเดียวกัน)

## ตรรกะ (ต้องตรงกับ MCP `atlas_pbl_unit_crosscheck` เป๊ะ)
- `pbl_pct = (com+think+problem+life+tech)/15*100`
- `gap = round(unit_pct - pbl_pct)`  (ลบ = PBL สูงกว่าเนื้อหา / บวก = เนื้อหาสูงกว่า PBL)
- จัดกลุ่ม:
  - `at_risk` (เร่งด่วน) = result==fail && unit<50
  - `content_strong` (รู้เนื้อหา/ประยุกต์อ่อน) = gap >= 10
  - `app_strong` (ทำได้/เนื้อหาอ่อน) = gap <= -10
  - `balanced` (สมดุล) = นอกนั้น

## UI ที่จะแสดง (ตามตัวกรองเทอม/ชั้น/ห้อง)
1. **กราฟ quadrant (ScatterChart ของ recharts)** — X = unit%, Y = pbl%, จุด = นักเรียน, hover เห็นชื่อ,
   ลากเส้นแบ่ง 4 ช่อง (ค่ากึ่งกลางหรือเกณฑ์), ลงสีตามกลุ่ม
2. **การ์ดสรุป 4 กลุ่ม** — ชื่อกลุ่ม + จำนวน + รายชื่อ (ใช้สี: เขียว=สมดุล, ฟ้า=content_strong, เหลือง=app_strong, แดง=at_risk)
3. **ตารางเด็กต้องดูแลพิเศษ** — fail หรือ unit<50 + หมายเหตุ (notes) + แนวพัฒนาตามกลุ่ม
4. **รายคน** — เลือกเด็ก → PBL 5 ด้าน คู่กับ K/P/A% + กลุ่ม + ทิศทางพัฒนา

## ทิศทางพัฒนาต่อกลุ่ม (ข้อความสำเร็จรูป — ใช้ชุดเดียวกับรายงาน .docx)
- at_risk: ช่วยเหลือเร่งด่วนรอบด้าน + ซ่อมเสริมความรู้พื้นฐานเข้ม + ประสานผู้ปกครอง + พิจารณา Special Care
- content_strong: เพิ่มกิจกรรมลงมือทำ/ทำงานกลุ่ม/ฝึกนำเสนอ (แปลงความรู้เป็นการปฏิบัติ)
- app_strong: เสริมความรู้เนื้อหา (อ่าน–เขียน–คำนวณ/มโนทัศน์หลัก) ควบคู่การลงมือทำ
- balanced: ต่อยอดด้วยโจทย์ท้าทาย/บทบาทผู้นำกลุ่ม

## Guard rails (ห้ามพลาด)
