# Handover: แก้ไข ai-chat validation — พีทตอบ follow-up ไม่ได้

**สำหรับ:** Cowork ที่รับผิดชอบ Supabase Edge Functions  
**วันที่:** 2026-03-23  
**สถานะ:** รอเสนอแผนก่อนลงมือแก้

---

## 1. ปัญหาที่พบ

เมื่อผู้ใช้ถามพีท (AI Consultant) **คำถามแรก** เช่น "ป.1/1 วิชาการคิดคำนวณ มีเด็กคนไหนต้องดูแลเป็นพิเศษ" → **ตอบได้ถูกต้อง** พร้อม [REF-13], [REF-15]

แต่เมื่อถาม **คำถามต่อเนื่อง** เช่น
- "ทำบันทึกแจ้งครูผู้สอนให้ทำแผน แก้ปัญหาให้เด็กกลุ่มนี้ด้วย"
- "มีคำแนะนำสำหรับครูที่สอนวิชา การคิดคำนวณ ป.1/1 หรือไม่"
- "ภาพรวมชั้นเรียนวิชา การคิดคำนวณ ป.1/1 เป็นอย่างไร"

→ ได้ข้อความ **"ไม่พบข้อมูลในระบบสำหรับตัวกรองที่เลือก หรือคำตอบมีการอ้างอิงที่ไม่ถูกต้อง กรุณาลองถามใหม่อีกครั้ง"** ทั้งหมด

**ข้อมูลมีอยู่จริง** — หน้า History แสดงข้อมูล ป.1/1 การคิดคำนวณ ได้ปกติ

---

## 2. สาเหตุ (จากการวิเคราะห์)

Validation ใน `ai-chat` ใช้ `validateAiChatOutput()` จาก `aiChatValidator.ts`  
กฎที่ทำให้ fail: ถ้าคำตอบมี **"claims"** ต้องมี `[REF-X]` อย่างน้อย 1 จุด มิฉะนั้นจะ fail ที่ `claims_without_refs`

ฟังก์ชัน `hasClaims()` ถือว่าคำเหล่านี้เป็น claim:
```
คะแนน, เฉลี่ย, จำนวน, คาบ, คน, ห้อง, ชั้น, วันที่
+ Mastery, Remedial, เศษส่วน X/Y, เปอร์เซ็นต์, วันที่ YYYY-MM-DD, ID XXXXX, ตัวเลข 2+ หลัก
```

ปัญหาคือ คำว่า **"ชั้น"** (ใน "ชั้นเรียน") และ **"คน"** ปรากฏในคำตอบทั่วไป เช่น
- "ภาพรวม**ชั้น**เรียน..."
- "แก้ปัญหาให้เด็ก**กลุ่ม**นี้" (ถ้ามีคำอื่นที่ match)
- "รูปแบบการสอนใน**ชั้น**เรียน"

→ ถือว่าเป็น claim ทั้งหมด → บังคับให้มี [REF-X] → ถ้า AI ไม่ใส่ REF ในทุกย่อหน้า → validation fail

## 3. ไฟล์ที่เกี่ยวข้อง (แก้เท่าที่จำเป็น)

| ไฟล์ | บทบาท |
|------|--------|
| `supabase/functions/_shared/aiChatValidator.ts` | ไฟล์หลักที่ต้องแก้ — โดยเฉพาะ `hasClaims()` และ/หรือ logic การบังคับ REF |
| `supabase/functions/ai-chat/index.ts` | เรียก `validateAiChatOutput` — **ไม่ต้องแก้** เว้นแต่มีเหตุผลชัดเจน |

**ห้ามแก้:**
- ระบบ REF format ([REF-1], [REF-2] ฯลฯ)
- การตรวจ REF ที่ไม่มีใน context
- การตรวจ ID invention, Remedial fraction
- ไฟล์อื่นที่ไม่เกี่ยวกับ validation

---

## 4. ขอบเขตที่ต้องการ

- **แก้เฉพาะ** logic ที่ทำให้คำตอบ follow-up ที่เป็น **คำแนะนำทั่วไป** ถูก reject โดยไม่จำเป็น
- **คง** การป้องกัน data leakage และการอ้างอิงผิด (REF/ID ที่ไม่อยู่ใน context)
- **ไม่** 放宽 กฎจน AI สร้างตัวเลขหรืออ้างอิง REF โดยไม่มีใน context ได้

---

## 5. สิ่งที่ต้องการจาก Cowork

### ก่อนลงมือแก้: ส่ง **แผนการแก้** มาให้ดูก่อน

เสนอแผนในรูปแบบนี้:

1. **สรุปสาเหตุ** ที่เชื่อว่าเป็น root cause (สั้นๆ)
2. **การแก้ที่เสนอ:**
   - แก้ไฟล์ใด บรรทัดประมาณไหน
   - เปลี่ยน logic อย่างไร (หรือเพิ่มเงื่อนไขอะไร)
3. **ผลกระทบที่คาดไว้:**
   - จะลด false positive (ตอบได้มากขึ้น) อย่างไร
   - จะไม่เพิ่ม false negative (ปล่อยคำตอบผิด/ไม่มีการอ้างอิงที่อันตราย) อย่างไร
4. **แนวทางทดสอบ** ที่จะใช้ตรวจหลังแก้

### หลังอนุมัติแผน: ค่อยลงมือแก้และ deploy

```bash
npx supabase functions deploy ai-chat
```

## 7. ตัวอย่างแนวทางแก้ (เป็นไอเดียเท่านั้น — ให้ Cowork เสนอแผนเอง)

แนวทางที่เป็นไปได้ (ไม่จำเป็นต้องทำตามทุกข้อ):

- แยกคำว่า "ชั้น", "ห้อง", "คน" ออกจากรายการ claim เมื่ออยู่ในบริบททั่วไป (เช่น "ชั้นเรียน", "ห้องเรียน", "กลุ่มคน")
- หรือใช้ pattern ที่เฉพาะเจาะจงกว่า เช่น ต้องอยู่ใกล้คำว่า "คะแนน", "จำนวน", "คาบ" ถึงจะนับเป็น claim
- หรือเพิ่มข้อยกเว้นสำหรับประโยคที่เป็นคำแนะนำ/สรุป (เช่น มี "แนะนำ", "ควร", "ครับ" ในประโยคใกล้เคียง) — ใช้ด้วยความระมัดระวัง

**หมายเหตุ:** ให้ Cowork วิเคราะห์และเสนอแผนที่เป็นไปได้มากที่สุด ไม่ต้องยึดตามตัวอย่างนี้
