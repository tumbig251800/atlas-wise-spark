**Typo ในโค้ด — แก้แล้ว ผ่าน PR:**

**Subject taxonomy rule (สำคัญ):**

`ภาษาอังกฤษ` กับ `ภาษาอังกฤษ KBW` เป็น **2 วิชาคนละตัว ห้ามรวม** — KBW เป็นโปรแกรม/หลักสูตรแยก เวลา group-by `subject` ในรายงานต้องปล่อยแยกบรรทัดเสมอ

### 3. Validation Layer Status

ฟอร์มสร้าง teaching log ([Step1General.tsx](../src/components/teaching-log/Step1General.tsx)) ใช้ `<Select>` dropdown bound กับ hardcoded `SUBJECTS[]` array — **ครูพิมพ์ subject เองไม่ได้** typo จะไม่หลุดเข้ามาใหม่ผ่านฟอร์มนี้

**ช่องโหว่ที่เหลือ (ยังไม่ได้แก้):**
- ไม่มี `CHECK constraint` หรือ trigger ฝั่ง Postgres
- 41 rows `คณิศาสตร์` เดิมน่ามาจาก bulk import / seed รุ่นแรก ไม่ใช่ผ่านฟอร์ม
- ถ้ามี import path อื่นในอนาคต (admin tool, CSV upload, API direct insert) อาจ inject ค่านอก whitelist ได้

**ข้อเสนอ defensive layer (ยังไม่ทำในเซสชันนี้):** เพิ่ม CHECK constraint บน `teaching_logs.subject` whitelist ค่าจาก `SUBJECTS[]` — ป้องกันทุก insertion path
