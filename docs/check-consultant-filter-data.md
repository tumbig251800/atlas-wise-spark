# การตรวจสอบ: พีทตอบ "ไม่พบข้อมูล" แม้กราฟมีข้อมูล

## สรุปผลการตรวจสอบ

### 1. สคริปต์เช็ค teaching_logs
สร้าง `scripts/check-teaching-logs.mjs` แล้วรัน — ได้ **0 แถว** เพราะใช้ anon key (ไม่มี session) ทำให้ RLS บล็อก ไม่สามารถยืนยันจำนวนข้อมูลจริงจาก script ได้

### 2. การวิเคราะห์จากโค้ด

**สาเหตุที่เป็นไปได้หลัก: ตัวกรองของ Consultant ไม่ตรงกับที่เลือกบน Dashboard**

| หน้า | แหล่งข้อมูล filter | พฤติกรรม |
|------|---------------------|----------|
| **Dashboard** | `filters` state + `persistFilters()` | เลือก ป.1/1 การอ่าน → บันทึกลง localStorage → กราฟแสดงตาม filter |
| **Consultant** | `contextFilter` แยกต่างหาก | โหลด `loadPersistedFilters()` ซึ่ง **return ค่าว่างเสมอ** → auto-init เป็น **คาบล่าสุดในระบบ** |

ดังนั้น:
- ตอนเปิด Consultant ตัวกรองจะเซ็ตเป็น **คาบล่าสุด** (เช่น ป.4/1 คณิตศาสตร์) ไม่ใช่ค่าที่เลือกบน Dashboard
- ถ้าไม่ได้กดเปลี่ยนตัวกรองบน Consultant เป็น ป.1/1 การอ่าน ค่า `filteredLogs` จะเป็นของชั้น/วิชาอื่น
- พอถามเรื่อง "ป.1/1 การอ่าน ภาพรวม" พีทได้ context คนละชุด → ตอบไม่ตรง หรือ validation ไม่ผ่าน → ขึ้น "ไม่พบข้อมูล..."

### 3. Validation layer (ai-chat)
ข้อความ "ไม่พบข้อมูลในระบบสำหรับตัวกรองที่เลือก หรือคำตอบมีการอ้างอิงที่ไม่ถูกต้อง" มาจาก `validateAiChatOutput` ไม่ผ่าน เช่น:
- ใช้ `[REF-x]` ที่ไม่มีใน context
- มีตัวเลข/เปอร์เซ็นต์แต่ไม่มีหลักฐานใน context
- รูปแบบ `[REF-x]` ไม่ถูกต้อง

## สิ่งที่ควรทำก่อนถามพีท

1. **เช็คตัวกรองบน Consultant**
   - ดูว่า dropdown วิชา / ชั้น / ห้อง ตรงกับที่ต้องการหรือไม่
   - ดูจำนวน "X คาบ" ด้านขวาของตัวกรอง ว่ามีข้อมูลหรือไม่

2. **เลือกตัวกรองให้ตรงกับคำถาม**
   - ถ้าจะถามเรื่อง ป.1/1 การอ่านและการเขียน → เลือก **วิชา: การอ่านและการเขียนเพื่อการสื่อสาร, ชั้น: ป.1, ห้อง: 1**
   - ตรวจให้แน่ใจว่า "X คาบ" > 0 ก่อนส่งคำถาม

3. **กรณีกราฟมีข้อมูลแต่ Consultant แสดง 0 คาบ**
   - ลองรีเฟรชหน้า Consultant
   - หรือไปที่ Dashboard เลือก filter เดิม แล้วกลับมาที่ Consultant อีกครั้ง (ถ้ามีการ sync filter ต่อมา)

---

## การแก้ไขที่แนะนำ (สำหรับนักพัฒนา)

- ให้ Consultant โหลด filter จาก `localStorage` (Dashboard) เมื่อเปิดหน้า แทนการ return ค่าว่างจาก `loadPersistedFilters()`
- หรือซิงค์ filter ระหว่าง Dashboard กับ Consultant เมื่ออยู่ในแอปเดียวกัน
