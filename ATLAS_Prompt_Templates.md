# 📘 ATLAS Prompt Templates — คู่มือสั่งงานวิเคราะห์ข้อมูล

> **วิธีใช้**: คัดลอก prompt ในกล่องที่ต้องการ แล้วส่งให้ AI ได้เลย
> สามารถแก้ไขส่วนที่อยู่ใน `[วงเล็บ]` ให้ตรงกับสิ่งที่ต้องการ

---

## 🗂️ หมวด A — ภาพรวมการสอน

### A1 · ภาพรวมทั้งหมด
```
วิเคราะห์ภาพรวมการสอนทั้งหมดในระบบ ATLAS ให้ครบถ้วน
ต้องการทราบ:
- จำนวน session ที่สอนทั้งหมด
- ค่าเฉลี่ย mastery score
- สัดส่วน Gap แต่ละประเภท (K-Gap / P-Gap / A-Gap / System-Gap)
- วิชาที่ได้คะแนนสูงสุดและต่ำสุด
สรุปออกมาเป็นไฟล์ Excel พร้อมกราฟ
```

### A2 · เปรียบเทียบช่วงเวลา
```
ดูภาพรวมการสอนย้อนหลัง [X สัปดาห์ / X เดือน]
โดยเปรียบเทียบ mastery score ระหว่างช่วงเวลาดังกล่าวกับช่วงก่อนหน้า
แสดงให้เห็นว่าคุณภาพการสอนดีขึ้นหรือแย่ลงอย่างไร
```

## 📚 หมวด B — วิเคราะห์รายวิชา

### B1 · เจาะลึกรายวิชา
```
วิเคราะห์เจาะลึกวิชา [ชื่อวิชา เช่น การอ่านและการเขียน]
ต้องการทราบ:
- แนวโน้ม mastery score รายเดือน
- Gap ที่พบบ่อยที่สุด
- session ที่อยู่ใน Red Zone (คะแนน ≤ 2.5)
- คำแนะนำในการปรับปรุง
```

### B2 · จัดอันดับทุกวิชา
```
เปรียบเทียบ mastery score ของทุกวิชาในระบบ
จัดอันดับจากสูงสุดถึงต่ำสุด
และแสดงว่าวิชาไหนมี Gap ประเภทใดเป็นหลัก
ทำออกมาเป็นตาราง Excel พร้อม chart
```

## ⏰ หมวด C — วิเคราะห์ตามช่วงเวลา

### C1 · ย้อนหลัง 2 สัปดาห์
```
วิเคราะห์ข้อมูลการสอนย้อนหลัง 2 สัปดาห์ (ตั้งแต่วันที่ [DD/MM] ถึง [DD/MM])
แสดงผลรายวัน และบอกว่าช่วงไหนมีคะแนนต่ำหรือสูงผิดปกติ
```

### C2 · แนวโน้มรายเดือน
```
ดูแนวโน้มการสอนรายเดือน ย้อนหลัง 3 เดือน
คำนวณ % การเปลี่ยนแปลงแต่ละเดือน
และระบุเดือนที่ผลลัพธ์ดีที่สุดและแย่ที่สุด
```

## 🎯 หมวด D — วิเคราะห์ Gap

### D1 · Gap Analysis รายวิชา
```
วิเคราะห์ Gap Analysis แยกตามประเภท K-Gap / P-Gap / A-Gap / System-Gap
บอกว่าแต่ละวิชามี Gap ใดบ้าง และ Gap ไหนเกิดซ้ำบ่อยที่สุด
เสนอแนวทางแก้ไขเป็นรายวิชา
```

### D2 · Red Zone Sessions
```
หา session ทั้งหมดที่อยู่ใน Red Zone (mastery score ≤ 2.5)
จัดกลุ่มตามวิชาและเดือน
และบอกว่าเกิดจาก Gap ประเภทใดบ้าง
```

---

## 🔴 หมวด E — TrendAlerts (แนวโน้มเสี่ยง)

### E1 · ตรวจสอบ TrendAlerts ทั้งหมด
```
วิเคราะห์ TrendAlerts จากข้อมูลในระบบ ATLAS
ต้องการทราบ:
- มีวิชา/ห้องไหนที่คะแนน 3 session ล่าสุด "ลดลงต่อเนื่อง" (Falling)
- มีวิชา/ห้องไหนที่ติด Red Zone (≤ 2.5) ทั้ง 3 session ล่าสุด
จัดกลุ่มตามวิชาและระดับชั้น
แสดงคะแนนแต่ละ session พร้อมบอกว่าควรแจ้งเตือนผู้บริหารหรือไม่
```

### E2 · ห้องเรียนติด Redzone
```
ตรวจสอบห้องเรียนที่เข้าข่าย TrendAlert ประเภท Redzone
(คือ mastery_score ≤ 2.5 ติดต่อกัน 3 session)
แสดงผลเรียงจากวิกฤตที่สุด พร้อมระบุครูผู้สอนและวันที่สอนล่าสุด
```

## 🟠 หมวด F — Strike System (Escalation)

### F1 · สรุป Strike ทั้งหมด
```
วิเคราะห์ Strike Counter ในระบบ ATLAS
แยกประเภทดังนี้:
- Strike 1 (Plan Fail) — มีกี่รายการ วิชาอะไรบ้าง
- Strike 2+ (Force Pivot) — มีกี่รายการ ต้องดำเนินการอะไร
- Resolved — มีกี่รายการที่แก้ไขแล้ว
จัดกลุ่มตาม scope (รายห้อง / รายนักเรียน)
และสรุปว่าวิชาไหนมีปัญหาซ้ำมากที่สุด
```

### F2 · Strike ที่ต้องดำเนินการ
```
ดึงรายการ Strike ที่ยัง Active และครบ 2 ครั้งขึ้นไป
ทั้งหมดนี้ต้องส่งเรื่องไปที่ใคร (ระดับชั้นเรียน / วิชาการ / ผู้บริหาร)
สรุปเป็นตาราง Excel พร้อมลำดับความเร่งด่วน
```

## 🔵 หมวด G — Diagnostic Color Status

### G1 · สรุปสถานะทั้งโรงเรียน
```
สรุปสถานะ Diagnostic ทั้งโรงเรียน แยกตามสี:
- RED (วิกฤต): มีกี่ session วิชาและห้องอะไรบ้าง
- ORANGE (เฝ้าระวัง): มีกี่ session
- YELLOW (Learning Curve): มีกี่ session
- BLUE (System-Gap): มีกี่ session ปัญหาเชิงระบบคืออะไร
- GREEN (สำเร็จ): มีกี่ session
เปรียบเทียบสัดส่วนและบอกว่าสถานการณ์โดยรวมน่าเป็นห่วงหรือไม่
```

### G2 · เจาะ RED และ ORANGE
```
เจาะ Diagnostic สี RED และ ORANGE
ระบุว่าแต่ละ session มี Gap ประเภทใด (K/P/A/System)
และวิชาไหนที่ปรากฏซ้ำมากที่สุดในกลุ่มสีวิกฤต
```

## 🚨 หมวด H — A2-Gap Referral Queue

### H1 · ดึง Referral Queue ทั้งหมด
```
ดึงรายการ Referral Queue ทั้งหมด (A2-Gap)
แสดง: หัวข้อ, วิชา, ชั้น/ห้อง, รายชื่อนักเรียน (ถ้ามี), Priority Level
เรียงจาก Priority สูงสุดก่อน
และสรุปว่ารายการไหนเร่งด่วนที่สุดที่ต้องส่งต่อผู้เชี่ยวชาญ
```

## 🏛️ หมวด I — รายงานผู้บริหาร (Director View)

### I1 · รายงานแจ้งเตือนรวม
```
สร้างรายงานแจ้งเตือนสำหรับผู้บริหาร โดยรวมข้อมูลจาก:
1. TrendAlerts (Falling + Redzone)
2. Strike Force Pivot ที่ยัง Active
3. A2-Gap Referral Queue ที่รอดำเนินการ
4. Session สี RED ทั้งหมด
ระบุว่าแต่ละปัญหาควรแจ้งใคร (ครู / วิชาการ / ผู้บริหาร)
และเสนอขั้นตอนการดำเนินการเป็นรายข้อ
ทำเป็น Excel พร้อมแท็บแยกแต่ละระดับการแจ้งเตือน
```

### I2 · Executive Alert Dashboard
```
สรุปภาพรวม Executive Alert Dashboard:
- จำนวน Alert แต่ละประเภทวันนี้
- เปรียบเทียบกับสัปดาห์ที่แล้ว
- ปัญหาที่วิกฤตที่สุด 3 อันดับแรก
- คำแนะนำเชิงกลยุทธ์สำหรับผู้บริหาร
```

### I3 · Executive Summary (Word/Excel)
```
สร้างรายงานสรุปสำหรับผู้บริหาร (Executive Summary) จากข้อมูลในระบบ ATLAS
เน้นประเด็นสำคัญ 3-5 จุดที่ควรให้ความสนใจ
พร้อมคำแนะนำเชิงกลยุทธ์
ทำออกมาเป็นไฟล์ Excel หรือ Word
```

## ⏱️ หมวด J — Alert ตามช่วงเวลา

### J1 · Alert ย้อนหลัง X วัน/สัปดาห์
```
ตรวจสอบ Alert ที่เกิดขึ้นใน [X สัปดาห์ / X วัน] ที่ผ่านมา
แสดงว่า Strike และ Redzone เพิ่มขึ้นหรือลดลงเมื่อเทียบกับช่วงก่อน
มีสัญญาณอะไรที่ต้องระวังเป็นพิเศษหรือไม่
```
